Security

Last updated: July 15, 2026

Scope: Daylo hosted API beta, public CLI, API spec, and agent skill

Daylo v0.1 is a small hosted API beta for smart-scale data, with an open-source CLI, public API spec, and agent skill. This page summarizes the operational security posture without replacing the Privacy Policy or Terms of Service.

Hosted beta

The production Daylo service is operated as a hosted API. Beta access is invite-first and waitlist-backed while the service is small.

No card required

The current beta is free and does not collect credit card or other payment information.

Open-source client surface

The CLI, API spec, and agent skill are public so users can inspect the integration contract and local client behavior.

Health-data boundary

Daylo retrieves and normalizes measurement data. It is not a medical, diagnosis, coaching, or emergency-response product.

§ 01Transport and Runtime

  • Daylo API traffic is served over HTTPS/TLS.
  • The hosted API runs on Cloudflare infrastructure, with persistent data stored in Turso.
  • Every API response includes X-Request-Id so incidents and support requests can be correlated without relying on prose logs.

§ 02Authentication and API Keys

  • Users sign in through supported passwordless flows such as Google sign-in or magic link.
  • The CLI uses a device authorization flow, then stores the generated API key in the local Daylo config file on the user's machine.
  • API keys are designed to be revocable and renewable if they are exposed.

§ 03Provider Tokens and Measurements

  • Daylo stores provider access tokens only to connect, sync, disconnect, and troubleshoot the user's selected providers.
  • Sensitive provider tokens are stored in encrypted form.
  • Measurement data is used to provide the API and CLI experience, not for advertising sales or AI model training.

§ 04Deletion and Disconnection

  • Disconnect one provider with daylo disconnect <provider>, which deletes stored tokens for that provider.
  • Delete stored measurements and provider tokens with DELETE /api/v1/me/data.
  • For account-level deletion, access, correction, or suspension requests, contact hello@pivop.jp.

§ 05Security Contact

Report suspected vulnerabilities, exposed keys, provider-token issues, or account-access issues to hello@pivop.jp. Include the request ID, affected account email, timestamp, and reproduction steps when available.

Please do not include full API keys, OAuth tokens, or unnecessary health data in reports. Redacted examples are enough to start triage.
