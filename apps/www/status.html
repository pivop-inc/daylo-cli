§ 02Provider Status

  • Withings: OAuth connect, token refresh, sync, latest reading, and normalization are verified end to end on the hosted production service.
  • Tanita Health Planet: OAuth connect, token exchange, sync, latest reading, and InnerScan normalization are verified end to end on the hosted production service.
  • Additional providers: not listed as supported until they pass the same contract and production checks.

§ 03Developer Surface

  • REST API under /api/v1, documented in the public spec.
  • JSON-first CLI for login, provider connect, sync, latest, list, and disconnect.
  • Agent skill for coding-agent sessions that need to sync or read weight data.

§ 04Known Limits

  • No SLA, paid plan, organization account, web dashboard, or webhook system in v0.1.
  • Beta behavior, provider coverage, and API details can change before a stable release.
  • Provider-side outages, API-policy changes, consent-screen constraints, or account limits can affect sync even when Daylo is healthy.

§ 05Where to Check

Use status.daylo.cc for the operations page once it is live. For current access questions, provider requests, or issue reports, use the links on Contact. For legal and data-handling details, read the Privacy Policy and Terms.
