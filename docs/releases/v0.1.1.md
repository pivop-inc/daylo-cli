# Daylo v0.1.1

Daylo v0.1.1 is a maintenance release that aligns the npm package, repository,
documentation, and release history around the `daylo-cli` name.

## Changes

- Bind saved CLI API keys to the origin where `daylo login` issued them, so
  `--api-url` and `DAYLO_API_URL` cannot forward a key to another origin.
- Require HTTPS for external API endpoints while retaining HTTP support for
  localhost loopback development and tests.
- Use the unscoped npm package name `daylo-cli` consistently.
- Use `bunx daylo-cli` as the canonical install and execution path.
- Expand the Japanese and English privacy policy and terms.
- Clarify measurement-data deletion, provider-token deletion, account-deletion
  requests, and limited legal-record retention.
- Simplify the public landing page while retaining direct links to the project
  and legal documents.
- Keep the existing CLI commands and `/api/v1` wire contract unchanged.

## Install

```sh
bunx daylo-cli@0.1.1 --help
```

## Supported Providers

- Withings
- Tanita Health Planet

## Known Limits

- Provider coverage is limited to Withings and Tanita Health Planet.
- The public repository contains the CLI, API contract, documentation, and
  agent-facing usage files. The hosted backend remains part of the Daylo service.
- Webhooks, organization features, dashboards, and consent-screen workflows are
  not included in v0.1.1.

## Release Verification

- `bun install --frozen-lockfile`
- `bun run check`
- `npm publish --dry-run --access public`
- `git diff --check`
- Production health and authentication smoke checks
