# Daylo v0.1.0

> Correction added on 2026-07-16: the npm package was renamed after this tag
> was created. The working install path for the renamed v0.1.0 package is
> `bunx daylo-cli@0.1.0`. This tag is retained as the historical source for the
> original `@pivop/daylo` release and does not exactly reproduce the renamed npm
> artifact.

Daylo v0.1.0 introduced a hosted smart-scale API and a JSON-first CLI for
developers and AI agents. It normalized provider measurements behind one REST
contract and returned machine-parseable JSON by default.

## Highlights

- JSON-first CLI for login, provider connection, sync, latest weight, and
  measurement history.
- Hosted API contract under `/api/v1`, documented in `docs/spec.md`.
- Withings support for OAuth connect, token refresh, sync, and normalized weight
  readings, verified end to end on production `daylo.cc` at release time.
- Tanita Health Planet support for OAuth, InnerScan sync, weight, and body-fat
  normalization, verified end to end on production `daylo.cc` at release time.
- Public CLI, API contract, documentation, and agent-facing usage files.

## Install

The currently available v0.1.0 package is installed with Bun:

```sh
bunx daylo-cli@0.1.0 --help
```

## Included CLI Commands

- `daylo login`
- `daylo connect withings|tanita`
- `daylo disconnect withings|tanita`
- `daylo sync`
- `daylo latest`
- `daylo list`

## Known Limits

- Provider coverage is limited to Withings and Tanita Health Planet.
- The public repo contains the CLI, API contract, docs, and agent-facing usage
  files; the hosted backend implementation remains part of the Daylo service.
- Webhooks, organization features, dashboards, and consent-screen workflows are
  out of scope for v0.1.0.

## Historical Verification

At the time of the original release, the public and backend checks passed, the
hosted API health endpoint responded successfully, and both launch providers had
been verified end to end. Public website surfaces may have changed since that
release; current behavior is documented in the repository and on `daylo.cc`.
