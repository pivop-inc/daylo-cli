# Daylo v0.1.0 Draft Release Notes

Draft only. Do not create a GitHub release or publish npm from this file without an explicit release approval.

## Summary

Daylo v0.1.0 introduces one API and a JSON-first CLI for smart scale data, built for developers and AI agents. It normalizes scale-provider measurements behind a single REST contract and a CLI that prints machine-parseable JSON by default.

## Highlights

- JSON-first CLI for login, provider connection, sync, latest weight, and measurement history.
- Hosted API contract under `/api/v1`, documented in `docs/spec.md`.
- Withings support verified end to end on production `daylo.cc`.
- Tanita Health Planet support verified end to end on production `daylo.cc`.
- Agent-oriented skill files for driving the CLI from Claude Code or Codex.
- npm install path for v0.1.0: `bunx @pivop/daylo`.

## What Is Included

- `daylo login` for browser-based sign-in and API key creation.
- `daylo connect withings|tanita` for provider OAuth.
- `daylo sync` to pull measurements from connected providers.
- `daylo latest` and `daylo list` for normalized weight data as JSON.
- Config stored at `~/.config/daylo/config.json` with `DAYLO_API_URL` and `--api-url` overrides.
- Public REST API contract for health checks, provider status, sync, latest/list weight data, provider disconnect, and data deletion.

## Known Limits

- v0.1.0 is installed from npm with Bun after the release publish gate completes.
- The public repo contains the CLI, API contract, docs, and agent skill. The hosted backend implementation remains part of the daylo service.
- Provider coverage is limited to Withings and Tanita Health Planet.
- Webhooks, organization features, dashboards, and consent-screen workflows are out of scope for v0.1.0.

## Verification Before Release

- Run `bun run check`.
- Run `bun test`.
- Run the CLI E2E suite against the mock backend.
- Smoke-test the CLI against production `daylo.cc` with a redacted test account.
- Confirm the README and `docs/spec.md` match this draft.
